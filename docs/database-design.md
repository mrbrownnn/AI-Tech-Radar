# Database Design

This document defines the MVP database schema for AI Tech Radar.

The database is PostgreSQL. UUID primary keys should be generated by the application or by PostgreSQL using an extension such as `pgcrypto`.

The current realtime design does not store raw source payloads. Collector output is normalized and upserted directly into `items`.

## Table: items

Purpose:

- Store normalized content from all sources.

Columns:

| Column | Type | Constraint | Description |
| --- | --- | --- | --- |
| `id` | UUID | Primary key | Internal item ID |
| `source` | VARCHAR(50) | Not null | Source name, for example `github`, `huggingface`, `arxiv`, or `rss` |
| `type` | VARCHAR(50) | Not null | Item type, for example `repository`, `model`, `dataset`, `space`, `paper`, or `news` |
| `title` | TEXT | Not null | Display title |
| `description` | TEXT | Nullable | Description, abstract, or summary |
| `url` | TEXT | Nullable | Canonical source URL |
| `tags` | JSONB | Not null default `[]` | Topics, labels, or task tags |
| `metadata` | JSONB | Not null default `{}` | Source-specific metrics and attributes |
| `published_at` | TIMESTAMP | Nullable | Source publish, release, or creation time |
| `created_at` | TIMESTAMP | Not null | Internal creation time |
| `updated_at` | TIMESTAMP | Not null | Internal update time |

Indexes:

| Index | Columns | Purpose |
| --- | --- | --- |
| `idx_items_source` | `source` | Filter by source |
| `idx_items_type` | `type` | Filter by content type |
| `idx_items_published_at` | `published_at` | Sort and filter by freshness |
| `idx_items_title` | `title` | Title lookup and deduplication support |

Recommended additional indexes:

| Index | Columns | Purpose |
| --- | --- | --- |
| `idx_items_source_type` | `source`, `type` | Common API and ranking filter |
| `idx_items_url` | `url` | URL deduplication |

## Table: scores

Purpose:

- Store ranking results for normalized items.

Columns:

| Column | Type | Constraint | Description |
| --- | --- | --- | --- |
| `id` | UUID | Primary key | Internal score ID |
| `item_id` | UUID | Foreign key to `items.id` | Ranked item |
| `popularity_score` | FLOAT | Not null | Popularity component |
| `activity_score` | FLOAT | Not null | Activity component |
| `recency_score` | FLOAT | Not null | Recency component |
| `relevance_score` | FLOAT | Not null | Relevance component |
| `final_score` | FLOAT | Not null | Weighted final score |
| `created_at` | TIMESTAMP | Not null | Score calculation time |

Recommended indexes:

| Index | Columns | Purpose |
| --- | --- | --- |
| `idx_scores_item_id` | `item_id` | Join scores to items |
| `idx_scores_final_score` | `final_score` | Query top ranked items |
| `idx_scores_created_at` | `created_at` | Query latest scoring run |

## Table: digests

Purpose:

- Store generated digest reports.

Columns:

| Column | Type | Constraint | Description |
| --- | --- | --- | --- |
| `id` | UUID | Primary key | Internal digest ID |
| `digest_date` | DATE | Not null | Reporting date |
| `content` | TEXT | Not null | Generated digest content, preferably Markdown |
| `channel` | VARCHAR(20) | Not null | Target channel, for example `telegram` or `email` |
| `created_at` | TIMESTAMP | Not null | Digest generation time |

Recommended indexes:

| Index | Columns | Purpose |
| --- | --- | --- |
| `idx_digests_digest_date` | `digest_date` | Query daily reports |
| `idx_digests_channel` | `channel` | Query by delivery channel |
| `idx_digests_created_at` | `created_at` | Query latest generated report |

## Table: delivery_logs

Purpose:

- Track delivery status for each digest and channel.

Columns:

| Column | Type | Constraint | Description |
| --- | --- | --- | --- |
| `id` | UUID | Primary key | Internal delivery log ID |
| `digest_id` | UUID | Foreign key to `digests.id` | Related digest |
| `channel` | VARCHAR(20) | Not null | Delivery channel |
| `status` | VARCHAR(20) | Not null | `pending`, `sent`, or `failed` |
| `error_message` | TEXT | Nullable | Failure details |
| `sent_at` | TIMESTAMP | Nullable | Successful delivery time |

Recommended indexes:

| Index | Columns | Purpose |
| --- | --- | --- |
| `idx_delivery_logs_digest_id` | `digest_id` | Join logs to digest |
| `idx_delivery_logs_channel` | `channel` | Query by channel |
| `idx_delivery_logs_status` | `status` | Find failed or pending deliveries |
| `idx_delivery_logs_sent_at` | `sent_at` | Delivery history lookup |

## Retention

Historical data should be retained for 12 months.

Retention policy:

- Keep `items`, `scores`, `digests`, and `delivery_logs` for 12 months by default.
- Archive or delete records older than the retention window.
- Keep Markdown exports when `EXPORT_MARKDOWN_REPORTS=true`.
